Instructions

This is a simple demo site for openFDA.com, where you may search for a Drug, Equipment or Food related item.

Medical professionals — such as Doctors, Nurses, Pharmacists & Pharmaceutical Lab Scientists — might use this search to find out drug related information. They’d be interested in finding out how 2 drugs interact & react with each other. They will also understand what all of the esoteric code names mean. Since there aren't any Food Labeling searches yet, consumers probably won’t have any reason to use this search form. They’d be interested in what is in their food when they go buy it at the store, but not what quality assurance checks were put in place by the FDA... before it reached the store. That seems to be what the “Enforcement reports” discuss. It would be nice to have a Food Labeling API search for consumers to use.

Since it feels overwhelming to read 10,000+ lines of deeply nested JSON data, we built a search tool that helps people see what’s in the JSON data feeds. This tool flattens nested JSON out into a nice clean easy-to-read list, for medical professionals & future web developers to use.

Now that this text is readable, we can start to see which data fields contain useful information & then decide how to display that data in a relevant manner. Charts & graphs look pretty, but if someone doesn't learn something from them when using them, then they are simply eye candy & don’t really need to be on the page. However, if they do teach the viewer something interesting, then the user has found value in visiting the website!

With our AJAX Search / JSON Analyzer Tool, we should have an easier time building charts like these: Example 1 & Example 2. Additional search fields can be found here.

Our AJAX Search / JSON Analyzer Tool also helps find out what data — in the API data feeds — would need to be cleaned up. It would be impossible to find that in the JSON data feeds, but it will show up on-screen as not looking correct. So it will show what HTML was incorrectly entered and/or copied & pasted into the FDA’s database.

Note: Long lists of array data — which are separated by commas & are over 2 lines long — will be displayed in a collapsed accordion. Those lists tend to be large lists of IDs and/or GUIDs, which nobody is going to want to simply scroll past. If they are shorter than 2 lines long, they will be displayed as regular text on the page & without an accordion, to make them easier to read.
